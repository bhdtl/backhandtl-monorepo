API: Matches
NEO.BET provides an API to request data of matches for your own banners.
In response to a request, the API always returns a list of matches.
These matches will be sorted by their popularity by default.
Request
The NEO.BET API is requested with a GET request.
Request parameters are sent as URL encoded query parameters.

Request without parameters
https://neo.bet/.sportsbet/program/matches
Request with parameters
https://neo.bet/.sportsbet/program/matches?sport=Football&region=ENG&league=Premier%20League
Query Parameters
The query parameters allow the request to be specified more precisely

API: Sports / Regions / Leagues / Teams
Request

In response to a request, the API always returns lists of filtered sports / regions / leagues / teams with their corresponding count in the current betable program.

Request without parameters
https://neo.bet/.sportsbet/program/parameter
Request with parameters
https://neo.bet/.sportsbet/program/parameter?sport=Football&region=DE&language=de

Requesting a full match
There are two ways to get full matches. You should choose the first way if you only need some specific full games and the second is if you really need every game.
Specific full
To request a full match you need to take some steps.
You have to start with our program API. You can find all the sports currently offered at https://neo.bet/.sportsbet/program/parameter
With this list you can request all regions for these sports at https://neo.bet/.sportsbet/program/parameter?sport=Football
You can use the sports and regions to request the leagues at https://neo.bet/.sportsbet/program/parameter?sport=Football&region=DE
With these 3 pieces of information you can request all games in this league at https://neo.bet/.sportsbet/program/matches?sport=Football&league=Bundesliga&region=DE&language=en This list already contains information on some markets, but in this case it is only needed to find the contest ids. The information about the markets can also be found in the final request.
Request the whole match at https://neo.bet/.sportsbet/program/matches?market=full&contestId=NEO|FB-1965757

General full
You can also use an import stream and pick every contest id from there. You can then proceed to step 5 from the steps above and request the full market for each contest id.

Ad-hoc requests
Ad-hoc requests can be used to immediately get a list of currently bettable matches in a specific sport, region or just the highlights that NEO.BET offers.

Example 1: German Bundesliga 
https://neo.bet/.sportsbet/program/matches?sport=Football&league=Bundesliga&region=DE&language=de&matchCount=10

Example 2: Top 10 Football matches
https://neo.bet/.sportsbet/program/matches?matchCount=10&sport=Football

Example 3: Recently updated Premier League matches
https://neo.bet/.sportsbet/program/matches?sport=Football&region=ENG&league=Premier%20League&team=Stoke%20City&matchCount=30&language=DE&updatedAfter=1520947417661
Import stream
To constantly import all matches or for example all matches of a specific sport into your system you can use our API with the updatedAfter parameter. Per default (without the matchCount Parameter) you would get them in chunks of 50 Matches beginning with the one that was updated the longest time ago. Repeating the requests you will always get the latest updates once you reach them. The next chunk can be requested by requesting it with the Parameter updatedAfter=<LATEST_TIMESTAMP_IN_PREVIOUS_UPDATE>

See the following Javascript Example which can easily be adopted to other languages.
Node.js Example
var request = require('request');

var REQUEST_URL = "https://neo.bet/.sportsbet/program/matches?updatedAfter="

function fetchMatches(updatedAfter) {
    request(REQUEST_URL + updatedAfter, function (error, response, body) {
        var lastUpdatedAfter = updatedAfter;
        if (response.statusCode === 200) {
            try {
                var matchList = JSON.parse(body);
                matchList.forEach(function (match) {
                    console.log(match);
                    // create or update the match in your database
                    // INSERT YOUR METHOD HERE
                    if (match.lastUpdated > lastUpdatedAfter)
                        lastUpdatedAfter = match.lastUpdated;
                });
            } catch (error) {
                console.log(error);
            }
        } else {
            console.log(error);
        }

        // get the next chunk or update 1 second later
        // be aware that 1 request per second is the maximum we allow
        setTimeout(function () { fetchMatches(lastUpdatedAfter) }, 1000);
    });
}

fetchMatches(0);
Linking

Deep Links to betslip
We recommend to use the contestId url to open the match detail:
https://neo.bet/?contestId=[CONTEST_ID]

You can also add bets directly to the betslip via several url parameters.
Format: ?betslip=compact&se=[CONTEST_ID]![BETTING_TYPE_KEY]![OUTCOME_KEY]

The parameter se (selection) can be added multiple times to add multiple bets.

Example:
https://neo.bet/de/Sportwetten/Fussball?betslip=compact&se=NEO|FB-2316997!Goal_RT_HC3W(0.0)!1&se=NEO|FB-2316994!Goal_RT_HC3W(0.0)!1

AffiliateId
You need to add an affiliateId to track users coming from your site. The affiliateId is a query parameter that has to be added to every url linking to neo.bet. 
Example: https://neo.bet/?affiliateId=yourId
Link Tool
Besides the link patterns described above you can also easily go to the https://neo.bet itself and use the link tool at the bottom of the site to create static links to where you are and the bets that you put into the betslip
